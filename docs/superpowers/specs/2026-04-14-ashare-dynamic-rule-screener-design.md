# A股规则选股动态调参设计

## 背景

当前 A 股规则选股日报存在两个问题：

1. 规则筛选把板块强度当成硬门槛，导致“技术形态命中很多，但最终结果为 0”。
2. 部分批量数据接口存在当日空表或延迟返回的情况，一旦换手率等辅助数据为空，整份报告会直接退化为“未筛出股票”。

这与目标用户不匹配。目标用户是有多年经验、偏稳健的 A 股老股民，更需要：

- 保留原始战法的骨架
- 在市场偏弱或数据不完整时，仍然提供有价值的观察池
- 明确说明哪些股票是完整命中、哪些是放宽命中、哪些只是技术候选
- 永远不要浪费当天的行情与积分，只返回“0 只”

## 目标

本设计要实现以下目标：

1. 保留你爸的核心战法，不把系统改成随机出票器。
2. 根据当天市场状态动态调整辅助阈值，而不是把所有参数钉死。
3. 报告永远分层输出，不允许再出现“技术形态命中 28 只，但最终只回 0 只”的产品结果。
4. 只有完整命中或轻度放宽命中的股票才自动并入 `STOCK_LIST`；技术候选池默认不自动并入。
5. 所有动态放宽都要在报告中明确说明，不能隐式放宽。

## 非目标

本设计不做以下事情：

1. 不试图做教科书式艾略特波浪识别，只做工程化的 ABC 式调整近似识别。
2. 不让 AI 决定筛选条件。AI 只负责复核、解释和排序，不参与硬规则筛选。
3. 不在第一版引入复杂机器学习打分模型，避免黑箱化。

## 核心原则

### 1. 核心条件固定

以下条件不做动态化，只允许长期手动配置：

- 前期已有明显涨幅
- ABC 式调整后再度转强
- 收盘重新站上 20 日均线
- `MA5 > MA10 > MA20`

这些条件构成策略骨架，必须稳定，不能随着市场波动每天漂移。

### 2. 辅助条件动态

以下条件允许根据当日市场状态动态调整：

- `min_volume_ratio`
- `min_turnover_rate`
- `min_sector_change_pct`
- `max_bias_ma5_pct`
- `abc_min_pullback_pct`
- `abc_min_rebound_pct`
- `abc_min_c_retention_ratio`

### 3. 结果永远分层，不再单层输出

日报结果必须拆成三层：

1. `完整命中`
完全满足你爸原始策略的股票。

2. `动态放宽命中`
当天市场偏弱或市场结构不支持，系统仅放宽辅助条件后仍满足核心结构的股票。

3. `技术候选池`
满足核心技术形态，但板块强度、换手率等辅助条件未通过的股票。该层用于人工精选，不自动加入自选池。

只要 `技术候选池` 非空，日报就不能返回“今日未筛出符合条件的A股股票”。

## 市场状态模型

系统每天先判断市场状态 `market_regime`，只分三档：

1. `strong` 强势日
2. `neutral` 中性日
3. `weak` 弱势日

### 输入因子

使用项目已能稳定获取的 A 股数据：

- 上证指数、深成指、创业板指涨跌幅
- 两市成交额
- 上涨家数 / 下跌家数
- 涨停家数 / 跌停家数
- 申万一级行业涨跌幅中位数
- 当天技术形态命中数量

### 判定逻辑

第一版使用简单、可解释的规则判定：

- 若指数整体偏强、上涨家数占优、板块中位数为正、涨停显著多于跌停，则为 `strong`
- 若指数分化但整体中性，则为 `neutral`
- 若指数偏弱、下跌家数显著占优、板块中位数偏弱，则为 `weak`

这部分只负责决定“允许放宽到什么程度”，不直接决定股票是否入选。

## 动态调参策略

### Level 0：原始策略

默认先用原始策略跑一次。该档位代表“完全按你爸原始战法”。

默认阈值：

- 前期累计涨幅 `>= 20%`
- 量比 `>= 1.5`
- 换手率 `>= 5.0%`
- 板块涨幅 `>= 2.0%`
- `MA5` 乖离率 `< 8.0%`

### Level 1：动态轻放宽

若 `Level 0` 结果太少，则根据 `market_regime` 放宽辅助阈值。

#### `strong`

强势日不主动放宽。最多只允许：

- 板块涨幅阈值从 `2.0%` 降到 `1.5%`

理由：强势日更容易追高，应该偏保守。

#### `neutral`

中性日允许轻放宽：

- 量比 `1.5 -> 1.3`
- 换手率 `5.0 -> 4.5`
- 板块涨幅 `2.0 -> 1.2`
- `MA5` 乖离率 `8.0 -> 8.5`
- ABC 容忍度轻微放松

#### `weak`

弱势日允许更明显放宽，但只限辅助条件：

- 量比 `1.5 -> 1.2`
- 换手率 `5.0 -> 4.0`
- 板块涨幅 `2.0 -> 0.8`
- `MA5` 乖离率 `8.0 -> 9.0`
- ABC 容忍度再轻放宽

### Level 2：技术候选池

若 `Level 1` 仍然为 0，则进入技术候选池输出：

- 保留上涨结构
- 保留 ABC 式调整
- 保留站上 `MA20`
- 保留均线多头排列
- 保留基本量能条件
- 取消“板块强度必须达标”的硬剔除

但报告里必须明确标记：

- 该股票属于 `技术候选池`
- 板块强度未达阈值
- 不自动加入自选池
- 仅供人工精选

## 避免空结果的策略

### 1. 永不空报告

日报输出顺序必须是：

1. `完整命中`
2. `动态放宽命中`
3. `技术候选池`

只有当三层都为空时，才允许输出“今日未筛出符合条件的A股股票”。

### 2. 空数据降级规则

对于关键批量数据，不能再简单视为“全部不满足”：

- `daily_basic` 当日空表时：
  - 优先尝试上一交易日缓存
  - 若仍为空，则降级为“换手率条件不可判定”
  - 股票仍可进入技术候选池，但报告里必须显式标注

- `sw_daily` 或板块映射为空时：
  - 不允许把全部股票直接清零
  - 进入技术候选池或动态放宽命中层时，板块条件退化为“信息不足，仅供人工判断”

### 3. 数据缓存策略

Tushare 批量数据继续落盘缓存，避免重复浪费积分：

- `trade_cal`
- `stock_basic`
- `daily`
- `daily_basic`
- `index_member_all`
- `sw_daily`

缓存原则：

- 非空结果才写缓存
- 空缓存自动忽略并重拉
- 历史日期结果必须可复用，不能因为 GitHub Actions 临时 runner 重新安装依赖就全部重拉

## 报告结构

日报结构改为：

1. `筛选档位`
2. `市场状态`
3. `动态调参说明`
4. `完整命中`
5. `动态放宽命中`
6. `技术候选池`
7. `AI复核`
8. `自选池同步`

### 每只股票要显示的字段

- 股票名 / 代码
- 所属板块与板块涨幅
- 现价 / `MA5` / `MA10` / `MA20`
- 量比 / 换手率
- 前高前累计涨幅
- 命中层级
- 规则说明
- 风险说明

### 明示项

若某票不是完整命中，必须显示原因：

- 因板块涨幅未达阈值，列入技术候选池
- 因换手率数据缺失，列入观察名单
- 因弱势日动态放宽，列入动态放宽命中

## 自选池同步规则

- `完整命中`：自动加入 `STOCK_LIST`
- `动态放宽命中`：自动加入 `STOCK_LIST`
- `技术候选池`：默认不自动加入，只在报告里提示人工确认

若未来需要，也可以加一个环境变量允许把技术候选池一并加入，但默认关闭。

  - 校验 workflow 默认环境与新变量

## 测试策略

必须覆盖：

1. 强势日 / 中性日 / 弱势日的动态参数生成
2. `Level 0 -> Level 1 -> Level 2` 的分级回退
3. `daily_basic` 空表时不会把全市场直接清零
4. 板块数据存在但不达阈值时，股票进入技术候选池而不是直接消失
5. 报告中必须出现动态调参说明与候选层级说明
6. 技术候选池默认不自动并入 `STOCK_LIST`

## 风险与边界

1. 动态调参不是为了“强行每天出票”，而是为了避免当天市场条件与原始战法错位时整份报告失去价值。
2. 若放宽过度，会稀释战法；因此只允许动态化辅助条件，不允许动态化核心结构条件。
3. 若数据源缺失严重，技术候选池仍可能质量下降，因此报告中必须清楚标记“数据缺失导致的降级”。

## 成功标准

实现完成后，应满足：

1. 不再出现“技术形态命中很多，但日报最终只返回 0”的情况。
2. 报告对每只股票都能解释其属于哪一层命中。
3. 用户能从日报中一眼看出：
   - 今天是强势/中性/弱势日
   - 哪些参数被动态调整了
   - 哪些票是完整命中，哪些只是观察候选
4. 在 `20260413` 这种历史案例上，系统至少能稳定输出非空结果。
